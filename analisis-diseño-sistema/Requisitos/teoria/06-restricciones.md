# Restricciones

## Definición

Las restricciones son limitaciones, condiciones o reglas que afectan el desarrollo, implementación o funcionamiento de un sistema.

A diferencia de los requisitos funcionales y no funcionales, las restricciones no describen lo que el sistema debe hacer ni cómo debe hacerlo, sino los límites dentro de los cuales debe operar.

Las restricciones pueden provenir de factores técnicos, económicos, legales, organizacionales o de tiempo.

---

## Importancia

Las restricciones permiten:

* Delimitar el alcance del proyecto.
* Identificar limitaciones desde el inicio.
* Reducir riesgos durante el desarrollo.
* Facilitar la toma de decisiones de diseño.

Ignorar las restricciones puede provocar retrasos, sobrecostos o soluciones imposibles de implementar.

---

## Tipos de Restricciones

### Restricciones Técnicas

Limitan las tecnologías, plataformas o herramientas que pueden utilizarse.

**Ejemplos:**

* El sistema debe desarrollarse utilizando Java.
* La base de datos debe ser PostgreSQL.
* La aplicación debe funcionar en navegadores modernos.

---

### Restricciones Económicas

Relacionadas con el presupuesto disponible.

**Ejemplos:**

* El costo total del proyecto no debe superar los $10,000.
* Deben utilizarse herramientas gratuitas o de código abierto.

---

### Restricciones de Tiempo

Relacionadas con fechas límite y cronogramas.

**Ejemplos:**

* El sistema debe estar operativo antes del inicio del próximo semestre.
* El proyecto debe completarse en 4 meses.

---

### Restricciones Legales

Provienen de leyes, normativas o regulaciones.

**Ejemplos:**

* Los datos personales deben cumplir con las leyes de protección de datos.
* El sistema debe conservar registros durante un período determinado.

---

### Restricciones Organizacionales

Surgen de políticas internas o decisiones de la organización.

**Ejemplos:**

* El sistema debe integrarse con el software ya existente.
* Solo ciertos empleados pueden acceder a funciones administrativas.

---

## Explicación Feynman

Las restricciones son las reglas del juego.

Imagina que te piden construir una casa.

Puedes diseñarla como quieras, pero existen límites:

* Presupuesto disponible.
* Tiempo para construirla.
* Terreno disponible.
* Leyes de construcción.

En software ocurre exactamente lo mismo.

Las restricciones establecen qué opciones son posibles y cuáles no.

---

## Ejemplo: Gestor de Turnos

### Restricciones Técnicas

* El sistema debe desarrollarse utilizando Node.js y Express.
* La información debe almacenarse en una base de datos relacional.

### Restricciones Económicas

* Deben utilizarse herramientas gratuitas durante la fase inicial del proyecto.

### Restricciones de Tiempo

* La primera versión debe estar disponible antes de la apertura del negocio.

### Restricciones Organizacionales

* Los empleados solo podrán visualizar y gestionar las citas asignadas.

### Restricciones Legales

* Los datos de los clientes deben almacenarse de forma segura.

---

## Diferencia entre Requisitos y Restricciones

### Requisito Funcional

Describe una función.

Ejemplo:

* El sistema debe permitir reservar turnos.

### Requisito No Funcional

Describe una característica de calidad.

Ejemplo:

* El sistema debe responder en menos de 2 segundos.

### Restricción

Describe un límite o condición.

Ejemplo:

* El sistema debe desarrollarse utilizando PostgreSQL.
