# Manuales de Usuario

## Definición

Un manual de usuario es un documento que explica de forma clara y sencilla cómo utilizar un sistema o aplicación.

Está dirigido al usuario final y tiene como objetivo ayudarle a comprender las funcionalidades del software sin necesidad de conocimientos técnicos.

---

# Importancia

Un manual de usuario permite:

* Facilitar el aprendizaje del sistema.
* Reducir errores durante su utilización.
* Disminuir la necesidad de soporte técnico.
* Mejorar la experiencia del usuario.
* Servir como guía de consulta.

Aunque una aplicación sea intuitiva, un manual ayuda a resolver dudas, explicar funcionalidades y orientar al usuario en tareas específicas.

---

# Objetivos

Un manual de usuario busca:

* Explicar cómo utilizar el sistema.
* Guiar al usuario paso a paso.
* Resolver las dudas más frecuentes.
* Mostrar las funcionalidades principales.
* Facilitar el aprendizaje de la aplicación.

---

# ¿Qué debe contener?

Un manual de usuario suele incluir:

* Introducción.
* Requisitos para utilizar el sistema.
* Inicio de sesión.
* Descripción de la interfaz.
* Funcionalidades principales.
* Preguntas frecuentes (FAQ).
* Solución de problemas.
* Información de contacto.

---

# Características de un Buen Manual

Un buen manual debe ser:

* Claro.
* Sencillo.
* Ordenado.
* Fácil de consultar.
* Actualizado.
* Orientado al usuario final.

Debe evitar lenguaje técnico innecesario y centrarse en explicar las tareas que el usuario desea realizar.

---

# Explicación Feynman

Imagina que compras una cafetera nueva.

No necesitas saber cómo funciona el circuito eléctrico interno.

Solo quieres saber:

* Cómo encenderla.
* Cómo preparar café.
* Qué hacer si aparece un mensaje de error.

El manual de usuario cumple exactamente esa función: enseñar a utilizar el producto, no explicar cómo fue construido.

---

# Estructura de un Manual de Usuario

## Introducción

Describe brevemente el propósito del sistema.

Ejemplo:

> El Gestor de Turnos permite reservar, modificar y cancelar citas de manera rápida desde una aplicación web.

---

## Requisitos

Explica qué necesita el usuario para utilizar el sistema.

Ejemplo:

* Navegador web actualizado.
* Conexión a Internet.
* Cuenta registrada.

---

## Inicio de Sesión

Describe el procedimiento para acceder al sistema.

Ejemplo:

1. Abrir la aplicación.
2. Introducir el correo electrónico.
3. Introducir la contraseña.
4. Pulsar **Iniciar sesión**.

---

## Funcionalidades

Explica paso a paso las operaciones principales.

### Reservar una cita

1. Seleccionar **Nueva cita**.
2. Elegir el servicio.
3. Seleccionar la fecha.
4. Elegir un horario disponible.
5. Confirmar la reserva.

---

### Cancelar una cita

1. Abrir **Mis citas**.
2. Seleccionar la cita.
3. Pulsar **Cancelar**.
4. Confirmar la operación.

---

### Consultar disponibilidad

1. Seleccionar un servicio.
2. Elegir una fecha.
3. Visualizar los horarios disponibles.

---

## Preguntas Frecuentes (FAQ)

### ¿Qué ocurre si olvido mi contraseña?

Utilice la opción **Recuperar contraseña**.

---

### ¿Puedo cancelar una cita?

Sí, siempre que la cita no haya sido atendida.

---

## Contacto

En caso de problemas, el usuario puede comunicarse con:

* Correo electrónico.
* Teléfono.
* Soporte técnico.

---

# Ejemplo: Gestor de Turnos

El manual del Gestor de Turnos permitiría a los clientes aprender a:

* Registrarse.
* Iniciar sesión.
* Reservar una cita.
* Reprogramar una cita.
* Cancelar una cita.
* Consultar el historial de citas.

Cada funcionalidad estaría acompañada de instrucciones claras y, cuando sea posible, capturas de pantalla.

---

# Uso de Imágenes

Los manuales de usuario suelen incorporar elementos visuales como:

* Capturas de pantalla.
* Flechas.
* Resaltados.
* Diagramas sencillos.

Las imágenes ayudan al usuario a identificar rápidamente los elementos de la interfaz y facilitan el aprendizaje del sistema.

---

# Relación con la Documentación Técnica

La documentación técnica responde a la pregunta:

> **¿Cómo está construido el sistema?**

El manual de usuario responde a la pregunta:

> **¿Cómo utilizo el sistema?**

Ambos documentos se complementan, pero están dirigidos a públicos diferentes.

---

# Buenas Prácticas

* Utilizar un lenguaje sencillo.
* Explicar una tarea por sección.
* Incluir imágenes cuando sea posible.
* Evitar tecnicismos innecesarios.
* Mantener el contenido actualizado.
* Organizar la información en un orden lógico.

---

# Relación con el Desarrollo Backend

Aunque el backend no suele redactar el manual de usuario, sí proporciona la información necesaria para documentar correctamente el funcionamiento del sistema.

Por ejemplo:

* Funcionalidades disponibles.
* Restricciones del negocio.
* Mensajes de error que puede recibir el usuario.
* Flujo de las operaciones.

---

# Conclusión

El manual de usuario es el documento que guía al usuario final en el uso del sistema.

Su propósito es enseñar de forma clara y práctica cómo utilizar las funcionalidades disponibles, facilitando el aprendizaje, reduciendo errores y mejorando la experiencia de uso de la aplicación.
